1. 홉바이홉 통신과 라우팅

홉바이홉(Hop-by-Hop) 통신: 패킷이 최종 목적지까지 도달하기 위해 여러 라우터를 거쳐가는 과정을 의미합니다.

라우팅 테이블: 송신지에서 수신지까지 가는 경로 정보가 담긴 리스트로, 다음 목적지 라우터의 정보를 포함합니다.

게이트웨이: 서로 다른 네트워크나 프로토콜 간의 통신을 가능하게 하는 '관문' 역할을 수행합니다.

2. IP 주소 체계 

IPv4 vs IPv6: 32비트 기반의 IPv4와 주소 부족 문제를 해결하기 위해 등장한 128비트(본문 오기 64비트) 기반의 IPv6가 있습니다.

클래스 기반 할당: 과거에는 A~E 클래스로 나누어 할당했으나, 버려지는 주소가 많다는 단점이 있었습니다.

주소의 구성: 네트워크 주소(처음 주소)와 브로드캐스트 주소(마지막 주소)는 장치에 할당할 수 없는 특수 용도로 사용됩니다.

3. 주소 부족 해결 및 보안 기술 

DHCP: 장치가 네트워크에 접속할 때마다 자동으로 IP 주소를 할당해주는 프로토콜입니다.

NAT (Network Address Translation): 사설 IP를 공인 IP로 변환하여 하나의 공인 IP로 여러 대의 장치가 인터넷을 쓸 수 있게 하며, 내부 IP를 숨겨 보안성도 높여줍니다.

IP 위치 정보: IP 주소를 통해 사용자의 대략적인 위치(도시, 구 단위) 추적이 가능합니다.
