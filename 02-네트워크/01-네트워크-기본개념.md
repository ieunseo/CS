## 2.1.2 네트워크 토폴로지와 병목 현상

### 네트워크 토폴로지(Network Topology)
- 노드(Node)와 링크(Link)가 어떻게 배치·연결되어 있는지를 의미
- 네트워크 성능, 확장성, 장애 전파 범위에 큰 영향

### 주요 토폴로지 종류

#### 트리(Tree) 토폴로지
- 계층형 구조
- 장점: 노드 추가·삭제가 쉬움
- 단점: 상위 노드 트래픽 집중 시 하위 노드 전체 영향

#### 버스(Bus) 토폴로지
- 하나의 중앙 통신 회선을 여러 노드가 공유
- 장점: 설치 비용 저렴, 구조 단순
- 단점: 스푸핑(Spoofing) 공격에 취약

**스푸핑(Spoofing)**
- 패킷을 가로채 의도하지 않은 노드로 전달되게 만드는 공격

#### 스타(Star) 토폴로지
- 중앙 노드(허브/스위치)에 모든 노드 연결
- 장점: 장애 탐지 용이, 패킷 충돌 적음
- 단점: 중앙 노드 장애 시 전체 네트워크 영향

### 병목 현상(Bottleneck)
- 특정 구성 요소가 전체 시스템 성능을 제한하는 현상
- 예: 트래픽 급증 시 서버 또는 네트워크 처리 한계 초과

## 2.1.3 네트워크 분류

네트워크는 **규모 기준**으로 분류됨

| 구분 | 설명 | 특징 |
|---|---|---|
| LAN | 근거리 통신망 | 전송 속도 빠름, 혼잡 적음 |
| MAN | 도시 단위 네트워크 | 중간 속도, 혼잡 증가 |
| WAN | 광역 네트워크 | 속도 느림, 혼잡 큼 |

## 2.1.4 네트워크 성능 분석 명령어

### 네트워크 병목 주요 원인
- 네트워크 대역폭
- 네트워크 토폴로지 구조
- 서버 CPU / 메모리 사용량
- 비효율적인 네트워크 구성

### 주요 명령어

#### ping
- 대상 노드로 ICMP 패킷 전송
- 네트워크 연결 여부 및 지연 시간 확인
- ICMP 차단 환경에서는 사용 불가

#### traceroute
- 목적지까지의 경로(홉) 추적
- 병목 구간 식별에 유용

#### 기타 도구
- `ftp` : 대용량 파일 전송 테스트
- `tcpdump` : 패킷 캡처
- `wireshark`, `netmon` : 네트워크 분석 툴

## 2.1.5 네트워크 프로토콜 표준화

### 네트워크 프로토콜
- 장치 간 통신을 위한 공통 규칙(인터페이스)

### 표준화 기관
- IEEE
- IETF

### 대표 예시
- **IEEE 802.3**
    - 유선 LAN(Ethernet) 표준
    - 제조사와 무관하게 상호 통신 가능
- **HTTP**
    - 웹 통신을 위한 대표 프로토콜
