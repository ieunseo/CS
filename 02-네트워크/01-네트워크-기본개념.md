# 네트워크(Network) 기본 개념 정리

## 1. 네트워크란?
네트워크(Network)란 여러 컴퓨터나 장치들이 서로 연결되어 데이터를 주고받을 수 있도록 구성된 통신 구조이다.

### 네트워크의 목적
- 데이터 및 정보 공유
- 자원 공유 (프린터, 서버, DB 등)
- 원격 통신 및 분산 처리
- 확장성과 효율성 향상

---

## 2. 네트워크 구성 요소

### 2.1 노드(Node)
네트워크에 연결된 모든 장치를 의미한다.
- 예: 컴퓨터, 서버, 스마트폰, 라우터

### 2.2 링크(Link)
노드 간 데이터를 전달하는 통신 경로
- 유선: 이더넷, 광케이블
- 무선: Wi-Fi, Bluetooth

---

## 3. 네트워크 범위에 따른 분류

### 3.1 LAN (Local Area Network)
- 제한된 지역(집, 사무실, 학교)
- 속도 빠름, 관리 용이

### 3.2 WAN (Wide Area Network)
- 넓은 지역(국가, 대륙)
- 대표 예: 인터넷

### 3.3 MAN (Metropolitan Area Network)
- 도시 단위 네트워크
- LAN과 WAN의 중간 규모

---

## 4. 네트워크 토폴로지 (Topology)
네트워크 장치들이 연결된 형태

### 4.1 스타(Star)형
- 중앙 장비(스위치)에 모든 노드 연결
- 장애 관리 용이

### 4.2 버스(Bus)형
- 하나의 케이블에 여러 노드 연결
- 구조 단순, 충돌 발생 가능

### 4.3 링(Ring)형
- 원형 구조로 연결
- 순차적 데이터 전달

---

## 5. OSI 7계층 모델
네트워크 통신 과정을 계층별로 나눈 표준 모델

| 계층 | 이름 | 역할 |
|---|---|---|
| 7 | 응용(Application) | 사용자 인터페이스 |
| 6 | 표현(Presentation) | 인코딩, 암호화 |
| 5 | 세션(Session) | 연결 관리 |
| 4 | 전송(Transport) | 신뢰성, 흐름 제어 |
| 3 | 네트워크(Network) | 경로 선택 |
| 2 | 데이터링크(Data Link) | 오류 검출 |
| 1 | 물리(Physical) | 전기적 신호 |

---

## 6. TCP/IP 4계층 모델

| 계층 | 주요 역할 |
|---|---|
| Application | HTTP, FTP, SMTP |
| Transport | TCP, UDP |
| Internet | IP, ICMP |
| Network Access | Ethernet |

---

## 7. 주요 프로토콜

### 7.1 TCP
- 신뢰성 보장
- 순서 보장
- 속도 느림

### 7.2 UDP
- 비신뢰성
- 속도 빠름
- 실시간 스트리밍에 적합

### 7.3 HTTP / HTTPS
- 웹 통신 프로토콜
- HTTPS는 암호화 제공

---

## 8. IP 주소

### 8.1 IPv4
- 32비트 주소
- 예: 192.168.0.1

### 8.2 IPv6
- 128비트 주소
- 주소 부족 문제 해결

---

## 9. 포트(Port)
하나의 IP에서 여러 서비스를 구분하기 위한 논리적 번호

| 포트 | 서비스 |
|---|---|
| 80 | HTTP |
| 443 | HTTPS |
| 22 | SSH |
| 3306 | MySQL |

---

## 10. DNS (Domain Name System)
도메인 이름을 IP 주소로 변환하는 시스템

- 사람이 이해하기 쉬운 주소 제공
- 예: google.com → IP 주소

---

## 11. 네트워크 장비

### 11.1 허브(Hub)
- 단순 신호 전달
- 충돌 발생 가능

### 11.2 스위치(Switch)
- MAC 주소 기반 전달
- 효율적 통신

### 11.3 라우터(Router)
- 네트워크 간 데이터 전달
- IP 기반 경로 설정
