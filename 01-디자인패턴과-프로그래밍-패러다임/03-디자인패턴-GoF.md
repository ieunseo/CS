## GoF패턴
**GoF 패턴의 정의**

객체지향 설계에서 반복적으로 발생하는 문제를
객체 간 관계로 해결하는 표준화된 설계 패턴

- 클래스/객체 단위
- 언어 독립적
- 재사용 가능한 구조
- 23개로 명확히 정의
- 생성 / 구조 / 행위 3가지로 분류 가능
> “어떤 객체를 어떻게 연결할 것인가”에 대한 이야기

## 비 GoF패턴
> GoF 책에는 없지만,실무·프레임워크·언어·아키텍처 레벨에서 반복적으로 쓰이는 설계 방식

- 생성/구조/행위로 깔끔하게 나누기 어려움
- 애플리케이션 구조 중심
- 프레임워크 의존적일 수 있음
- 언어 특화 가능

```
GoF 패턴 = 객체 설계 문제 해결 공식
비-GoF 패턴 = 시스템 설계 관점 
```

---
| 구분 | GoF 패턴 | 비-GoF 패턴 |
|---|---|---|
| 설계 범위 | 객체 / 클래스 | 시스템 / 레이어 / 아키텍처 |
| 초점 | 객체 간 관계 | 역할 분리, 책임 배치 |
| 분류 | 생성 / 구조 / 행위 | 분류 체계 없음 |
| 언어 의존성 | 낮음 | 높을 수 있음 |
| 시험 출제 | 매우 높음 | 개념 질문 위주 |
---

# 생성(Creational) 패턴

### 1. 추상 팩토리 (Abstract Factory)
- 관련 있는 객체들의 집합을 생성하기 위한 인터페이스 제공
- 구체적인 클래스는 명시하지 않음
- 팩토리 패턴의 확장 개념

**핵심 키워드**
- 객체 생성 추상화
- 제품군 단위 생성
- 구현과 생성 분리

---

### 2. 빌더 (Builder)
- 복잡한 객체의 생성 과정을 단계별로 분리하는 패턴
- 동일한 생성 과정으로 다양한 표현 결과 생성 가능

**핵심 키워드**
- 객체 생성 과정 캡슐화
- 생성 순서 제어
- 가독성 향상

---

### 3. 프로토타입 (Prototype)
- 기존 객체를 복사(clone)하여 새로운 객체 생성
- 생성 비용이 큰 객체에 적합

**핵심 키워드**
- 객체 복제
- new 연산자 최소화
- 성능 최적화

---

# 구조(Structural) 패턴

### 4. 어댑터 (Adapter)
- 호환되지 않는 인터페이스를 연결해주는 패턴
- 기존 코드를 수정하지 않고 재사용 가능

**핵심 키워드**
- 인터페이스 변환
- 호환성 확보
- Wrapper 역할

---

### 5. 브리지 (Bridge)
- 추상화와 구현을 분리하여 독립적으로 확장 가능
- 기능 계층과 구현 계층 분리

**핵심 키워드**
- 기능 / 구현 분리
- 확장성 강화
- 다형성 활용

---

### 6. 컴포지트 (Composite)
- 객체들을 트리 구조로 구성하여 단일 객체처럼 다룸
- 부분-전체 계층 표현

**핵심 키워드**
- 트리 구조
- 일관된 인터페이스
- 재귀적 구성

---

### 7. 데코레이터 (Decorator)
- 객체에 동적으로 기능을 추가하는 패턴
- 상속 대신 조합 사용

**핵심 키워드**
- 기능 확장
- 런타임 확장
- 상속 대체

---

### 8. 퍼사드 (Facade)
- 복잡한 서브시스템에 대해 단순한 인터페이스 제공
- 클라이언트와 내부 구조 분리

**핵심 키워드**
- 진입점 단순화
- 의존성 감소
- 서브시스템 캡슐화

---

### 9. 플라이웨이트 (Flyweight)
- 공유 가능한 객체를 재사용하여 메모리 사용 최소화
- 동일한 객체를 여러 번 생성하지 않음

**핵심 키워드**
- 객체 공유
- 메모리 절약
- 내부 상태 / 외부 상태 분리

---

# 행위(Behavioral) 패턴

### 10. 책임 연쇄 (Chain of Responsibility)
- 요청을 여러 객체를 따라 전달하며 처리 여부 결정
- 요청자와 처리자 분리

**핵심 키워드**
- 요청 전달
- 결합도 감소
- 처리 책임 분산

---

### 11. 커맨드 (Command)
- 요청을 객체로 캡슐화하여 실행, 취소, 로그 처리 가능

**핵심 키워드**
- 요청 객체화
- 실행 취소 (Undo)
- 큐 / 로그 처리

---

### 12. 중재자 (Mediator)
- 객체 간 직접 통신을 제거하고 중재 객체를 통해 상호작용

**핵심 키워드**
- 객체 간 결합도 감소
- 중앙 통제
- 복잡한 상호작용 관리

---

### 13. 메멘토 (Memento)
- 객체의 상태를 저장했다가 이전 상태로 복구하는 패턴

**핵심 키워드**
- 상태 저장
- Undo 기능
- 캡슐화 유지

---

### 14. 상태 (State)
- 객체의 내부 상태에 따라 행위가 변경되도록 캡슐화

**핵심 키워드**
- 상태 기반 행위
- 조건문 제거
- 객체 분리

---

### 15. 템플릿 메서드 (Template Method)
- 알고리즘의 전체 구조는 상위 클래스에
- 세부 구현은 하위 클래스에서 정의

**핵심 키워드**
- 알고리즘 뼈대
- 코드 재사용
- 상속 기반 확장

---

### 16. 비지터 (Visitor)
- 객체 구조는 그대로 두고 새로운 기능을 외부에서 추가

**핵심 키워드**
- 기능 확장
- 객체 구조 변경 없음
- 이중 디스패치

---

### 17. 인터프리터 (Interpreter)
- 문법 규칙을 클래스로 표현하여 해석 수행
- 간단한 언어 해석기에 사용

**핵심 키워드**
- 문법 트리
- 표현식 해석
- DSL 구현

---

# 요약

- **생성 패턴**: 객체 생성 방식 캡슐화
- **구조 패턴**: 클래스 / 객체 구조 설계
- **행위 패턴**: 객체 간 책임과 알고리즘 분리
